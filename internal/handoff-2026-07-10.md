## What this is

GitBook documentation for the Commonwealth of Learning (COL) Frugal AI initiative. Public content lives under `docs/` with navigation in `docs/SUMMARY.md`; internal maintainer material lives under `internal/` and is never linked from public navigation. The knowledge base is developer and technical first, with stakeholder framing on dedicated pages. `AGENTS.md` at the repo root is committed guidance for AI agents working in the repository.

## The model (the spine)

A Frugal AI system is a stack of substitutable, optional layers:

- **Infrastructure → Inference → Orchestration → Application**, with the **Gateway as the sovereignty envelope** for model egress.
- **Governance has two homes** (landing-page claim since 2026-07-10): the gateway governs model egress; the **agent loop** — governed where it runs — is the second home, because tool and MCP egress does not pass the gateway. Agents have **three governance surfaces**: local actions, model egress, tool egress.
- An **agent is an Application subtype**, not a layer. The agent loop (harness) is defined on the orchestration layer, where a "Tools by governance profile" table maps each tool class to its worked example.
- The **frugal floor** (Infrastructure + Inference + Application, gateway local-only, no orchestration) is the local chat build.
- The landing carries the **Three goals, three practices** hexagon: goals Frugality, Sovereignty, Openness (solid dots) opposite the practices that make them real — Local first, Teacher-in-the-loop, Capacity building (ring dots). All seven hexagon SVGs are generated by `internal/tools/generate_hexagons.py`; never hand-edit them. Six single-node variants are embedded in "Where this fits" sections on their pages.

The first build is the Local AI chat service: Mac mini 24 GB → Ollama → **Gemma 4 12B** (default since 2026-06-12; Qwen3.5-9B is the smaller alternative) → Open WebUI. The maths through-line runs local chat → math tutor → AI gateway → Manim animator; the Manim guide follows a staged, skill-guided pattern adapted from Math-To-Manim's HERMES workflow (storyboard approved before code, render inspected against explicit gates).

## Key decisions a maintainer must know

- **Principles page is internal** (`internal/frugal-ai-principles.md`); the public agentic framing home is `docs/concepts/application-layer.md`, which also carries the delegation-capacity argument.
- **Tier vocabulary is goals & practices** — not "pillars", "ends", or "principles". Registered in the naming registry with the node names.
- **Risk tiers use the annex vocabulary**: Tier 1 (learner-facing, approval before release), Tier 2 (teacher-only drafts, post-hoc audit), Tier 3 (automated with logging).
- **Model cards carry a `_Sources checked: YYYY-MM-DD._` line** in their Source confidence block (audited). Dates reflect when values were verified, not when the file was last touched.
- **Alternatives are prose references, not cards**: Tau and Pi in the coding-agent guide; Pi again as the harness example on the orchestration layer. The `Harness:` role label, a Pi card, and the skill registry are all deferred to plan-008.
- **Inclusion, the AI gap, and AI literacy enter as practice-linked content**, never as values pages: the landing's cited problem statement, the localisation section's inclusion thread, the AI-literacy glossary entry plus review-step lines in the Tier 1 guides.
- **Reference page is a faithful adaptation of the annex v2 stable content**; v2 proposals wait in `internal/annex-v2-proposed-changes.md`.
- **Port policy**: upstream defaults, every host port registered. **Operations policy**: every operable component in the operations overview, runbook or marked further work.
